# 3:4 图文模式升级规划

## 背景

当前 `xiaoliang-html-ppt-skill` 默认生成 16:9 横版 HTML PPT，主要服务于录屏讲解、教程演示和横屏翻页展示。现有能力已经覆盖：

- 3 种视觉风格：小亮手绘实验室、圆润孟菲斯、粗黑孟菲斯。
- 6 种基础布局：`image`、`imageWide`、`duo`、`imageStep`、`text`、`steps`。
- 录屏交互：键盘翻页、滚轮翻页、概览模式、全屏、hash 深链、图片悬停放大。

下一步希望增加 3:4 竖版 HTML 图文模式，用于制作更适合图文发布的卡片式内容。

## 核心目标

在保留 16:9 横版 PPT 能力的基础上，新增 3:4 竖版图文卡片模式。

这不是简单把 `aspect-ratio: 16 / 9` 改成 `3 / 4`，而是增加一种新的内容组织方式：

- 16:9 横版：适合讲课、录屏、界面演示。
- 3:4 竖版：适合图文发布、知识卡片、小红书/公众号配图、朋友圈图文。

默认仍然保持 16:9。只有用户明确说“3:4”“图文”“竖版”“卡片”“小红书”等需求时，才启用 3:4 模式。

## 模式命名

建议把输出模式抽象成 `output mode`：

- `deck`：16:9 横版课件模式，默认模式。
- `cards`：3:4 竖版图文模式。

用户触发方式：

- “做成 16:9 HTML PPT” -> `deck`
- “做成 3:4 图文版” -> `cards`
- “适合小红书图文” -> `cards`
- “竖版卡片” -> `cards`
- “不要横屏录屏，用图文发布比例” -> `cards`

## 内容策略差异

### 16:9 横版课件模式

适合讲解型内容：

- 一页一个讲解点。
- 可以放较长的解释文字。
- 适合左文右图、步骤加截图、大图界面演示。
- 详细教程可拆成 30-50 页。
- 页面依赖讲述者口播补充上下文。

### 3:4 竖版图文模式

适合发布型内容：

- 一页就是一张可独立阅读的图文卡片。
- 标题更像观点、判断或金句。
- 正文更短，但信息密度更高。
- 截图更少、更大、更聚焦。
- 更重视封面、转折页、总结页。
- 推荐 8-20 页，不建议机械拆成 30-50 页。
- 页面本身应尽量不依赖口播也能读懂。

## 3:4 视觉原则

3:4 模式继续继承现有三种风格，但需要竖版适配：

- 小亮手绘实验室：更像手绘知识卡片、笔记截图、图文教程。
- 圆润孟菲斯：更适合轻松图文、品牌故事、卡通感内容。
- 粗黑孟菲斯：更适合强标题、爆款感、观点型图文。

竖版共同规则：

- 标题放大，但控制行数，避免超过 2-3 行。
- 信息从上到下组织，少用横向左右分栏。
- 优先使用上文下图、上图下文、纵向步骤、总结卡片。
- 截图只展示最关键区域，避免整屏 UI 缩到看不清。
- 每页底部保留页码或系列标识，方便图文连载。

## 模板设计方案

### 比例控制

建议使用 body class 控制比例：

```html
<body class="style-xiaoliang-lab aspect-16x9">
<body class="style-xiaoliang-lab aspect-3x4">
```

CSS 结构：

```css
body.aspect-16x9 .slide {
  aspect-ratio: 16 / 9;
}

body.aspect-3x4 .slide {
  aspect-ratio: 3 / 4;
}
```

默认模板使用 `aspect-16x9`，保证现有行为不变。

### 3:4 专用布局

建议新增或适配以下布局：

- `coverVertical`：竖版封面，强标题、短副标题、主视觉。
- `quoteCard`：观点/金句卡，适合传播型单页。
- `stackedImage`：上文下图或上图下文，适合截图讲解。
- `verticalSteps`：竖向步骤卡，适合教程拆解。
- `comparisonVertical`：上下对比或轻量左右对比。
- `summaryCards`：结尾总结卡，适合行动清单。
- `screenshotFocus`：单截图重点标注页，避免界面信息过小。

3:4 模式不建议直接复用 `imageWide`，因为横版大图布局会在竖版中变得拥挤。

## 验证标准

基础验证：

- `quick_validate.py` 通过。
- 模板脚本语法检查通过。
- 默认打开仍是 16:9。
- 切换到 `aspect-3x4` 后页面比例正确。
- 键盘、滚轮、概览、hash 深链仍正常。
- 图片 hover zoom 仍正常。

3:4 专项验证：

- 标题不溢出。
- 主要文字在竖版中可读。
- 截图没有被缩到无法辨认。
- 不出现横版左右分栏硬塞进竖版的问题。
- 一组 8-20 页卡片能形成完整图文发布路径。

## 分阶段执行计划

### 阶段 1：文档升级

目标：让 skill 明确知道何时生成 16:9，何时生成 3:4。

任务：

- 更新 README 的能力介绍和触发示例。
- 更新 SKILL.md 的输出模式判断。
- 更新 authoring workflow 和 design system。
- 保持模板代码暂不大改。

验收：

- 文档能清楚解释 `deck` 和 `cards` 的区别。
- 用户说“图文/小红书/3:4/竖版”时，skill 会选择 3:4。

### 阶段 2：模板最小实现

目标：让模板真正支持 3:4。

任务：

- 添加 `aspect-3x4`。
- 添加 4 个 3:4 核心布局。
- 添加示例 slide。
- 验证 16:9 不回退。

验收：

- 修改 body class 即可在 16:9 和 3:4 间切换。
- 示例页在 3:4 下不溢出。
- 现有横版布局仍可用。

### 阶段 3：导出图片友好

目标：让 3:4 模式更适合实际发布。

任务：

- 增加截图导出说明或脚本。
- 考虑 Playwright 批量导出 PNG。
- 规划输出目录，例如 `exports/cards/*.png`。

验收：

- 一组 3:4 HTML slides 可以稳定导出成图片。
- 图片尺寸、边距、字体清晰度适合图文平台发布。

## 暂不做的事情

当前不建议马上扩展任意比例，避免模型选择空间过大。

暂不做：

- 1:1
- 4:5
- 9:16
- 任意自定义比例
- 复杂导出面板

先把 `16:9 deck` 和 `3:4 cards` 两种模式做扎实。

## 推荐下一步

下一步先执行阶段 1：文档升级。

完成后再进入阶段 2：模板最小实现。这样风险较低，也能保证 skill 的行为先被定义清楚，再写模板代码。
