# 作者工作流

## 1. 阅读并拆解文章

阅读来源内容时，为每个段落或小节记录：

- 核心观点是什么。
- 它解决了观众的哪一个困惑。
- 是否有具体例子、界面或操作。
- 需要哪类截图或插图辅助理解。
- 观众看完这一页应该记住什么。

不要把详细教程压缩成泛泛的三五个要点。原文如果认真解释了一个概念，就应该拆成多页讲清楚。

## 2. 建立图片清单

对每张图片记录：

- 文件名。
- 类型：概念图、截图、细节图、流程图、封面图、收尾图。
- 画面里能看到什么界面或内容。
- 最适合放在哪一页。
- 是否需要悬停放大。

图片多时，先做 contact sheet 或逐张检查。不要在没看图的情况下安排页面。

## 3. 决定输出模式和页数

默认选择 `deck`，除非用户明确提到 3:4、竖版、图文、卡片、小红书、公众号配图或社交发布。

`deck` 页数参考：

- 1 页封面。
- 1 页学习结果。
- 1-2 页解释为什么重要。
- 每个主要概念 2-4 页。
- 每个关键界面或控件 1 页。
- 1-2 页总结和行动。

详细教程通常需要 30-50 页。

`cards` 页数参考：

- 1 张封面卡。
- 1 张问题或钩子卡。
- 4-12 张概念/流程卡。
- 1-3 张总结或行动卡。

详细文章通常变成 8-20 张卡片。不要机械地把每个段落都切成一张卡，而要重新组织成清楚的图文阅读路径。

## 4. 写页面文案

全部文案优先使用中文。必要的代码、文件名、快捷键、布局名和 URL 参数可以保留英文。

一页好的小亮实验室 PPT 通常包含：

- 一个 section pin。
- 一个短角色标签。
- 一个清晰判断式标题。
- 一段 lead 或 3-5 个具体卡片/清单。
- 一张直接支撑当前观点的截图或视觉。

避免空话：

- “提升效率”
- “方便协作”
- “更好管理”

替换成具体教学表达：

- “PR 页面会显示哪些文件被改了，绿色是新增，红色是删除。”
- “Settings > Pages 里选好分支和目录后，GitHub 会生成 github.io 地址。”
- “图文卡片只展示当前要讲的按钮，不塞完整桌面截图。”

`cards` 模式下，每张卡都要不依赖口播也能读懂：

- 标题写观点，不只写话题。
- 正文比横版更短。
- 每张卡保留 2-4 个强信息点。
- 卡片之间形成“问题、方法、例子、行动”的阅读路径。
- 避免使用“上一页提到”这类依赖前文的表达。

## 5. 放置图片

图片要放在观众需要它的地方：

- 定义页：概念图。
- 操作页：真实截图。
- 总结页：流程图或结构图。
- 收尾页：人物、社区、开源或行动视觉。

如果截图里有细节，应给它足够大的右侧区域或单独一页。不要把关键截图缩得看不清。

截图卡片要保持原图比例。不要用固定高度让宽截图上下出现大块空白。

“步骤 + 截图”页面优先使用纵向步骤条。长中文标题和说明需要足够宽度自然换行。

`cards` 模式下，截图要更聚焦。完整桌面截图在 3:4 里通常太小，优先裁切到关键控件、关键区域或一张大截图配短说明。

3:4 封面卡不能把主视觉压到底部。标题、lead 和主视觉应该形成一组连续阅读区域；如果中间出现大块空白，要调整布局、放大主视觉或改用更适合竖版的裁切图。封面图允许轻微裁切来填满视觉区，但不能切掉主要人物、界面或关键概念。

## 6. 实现

优先使用数据驱动的单文件 HTML：

- 定义 `slides = [...]`。
- 用 JavaScript 渲染布局模板。
- CSS token 放在顶部。
- 交互脚本放在底部。
- 图片使用相对路径。

可从 `assets/template.html` 开始改。

横版布局：

- `image`：左文右图。
- `imageWide`：较窄文字 + 大截图。
- `duo`：文字 + 两张上下堆叠图片。
- `imageStep`：纵向步骤条 + 一张截图。
- `text`：标题 + 信息卡片或清单。
- `steps`：无截图步骤页。

竖版布局：

- `coverVertical`：3:4 封面卡。
- `quoteCard`：观点或金句卡。
- `stackedImage`：文字 + 聚焦截图。
- `verticalSteps`：竖向步骤卡。

## 7. 验证

最终交付前做浏览器检查：

生成 HTML 后先跑 `scripts/validate-generated-html.mjs`，再做浏览器检查。不要只靠肉眼确认页面存在。

- 统计页数。
- 检查所有引用图片存在并加载成功。
- 打开 `#/3` 这类深链，确认能跳到正确页面。
- 确认比例正确：`deck` 为 16:9，`cards` 为 3:4。
- 检查关键文字和图片不溢出。
- `cards` 每页都能独立阅读，没有横版布局硬塞进竖版。
- 3:4 封面没有标题和主视觉分离过远的问题，中段不能出现明显空洞。
- 测试键盘导航。
- 测试滚轮导航。
- 测试概览模式。
- 测试图片悬停放大。
- `cards` 模式下确认放大图片限制在当前 3:4 页面内。

最终回复应报告文件路径、页数、图片覆盖情况和验证结果。
